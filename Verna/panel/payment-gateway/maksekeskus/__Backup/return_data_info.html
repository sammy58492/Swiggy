Parameter Constraints Description
shop UUID Shop ID
transaction UUID Transaction ID
status Possible values: COMPLETED APPROVED CANCELLED EXPIRED REFUNDED PART_REFUNDED Transaction status. Values are explained here.
amount Positive number with 2 decimal places Transaction amount
currency Transaction currency
reference Merchant’s order reference, ie. to associate payment data in MK and merchant’s systems
customer_name Customer’s name who made the payment
message_time Message timestamp
message_type Message type is always payment_return for payment confirmation messages.
